# TICKET-026

실시간 축과 배치 축 분리 리팩토링.

## 1. 배경

현재 StockMaster는 다음 세 축의 정보를 한 제품 안에서 함께 보여준다.

- 장중 실시간/준실시간 판단 축
- 장마감 기준 일간 배치 축
- 다음 거래일 진입 계획 축

문제는 이 세 축이 UI에서 충분히 분리되지 않아, 사용자가 다음과 같이 오해할 수 있다는 점이다.

- 전일 종가 기준 시장국면을 현재 장 분위기로 오해
- 장마감 배치 기준 추천 점수를 현재 시점의 실시간 추천으로 오해
- 다음 거래일 진입용 포트폴리오 계획을 오늘 장중 액션처럼 오해
- 장중 데이터가 약한 상태인데도 화면 문구상으로는 현재 장 상황을 확정적으로 말하는 것처럼 보임

이 문제는 단순 문구 수정이 아니라, 제품 전반의 `시간 기준(ref time axis)`과 `용도(use axis)`를 구조적으로 분리해야 해결된다.

이 티켓은 그 리팩토링의 기준 문서다.

## 2. 왜 이 작업이 우선순위 1인가

이 작업이 먼저 끝나야 하는 이유:

- 사용자 신뢰도에 직접 영향
- 장중 오해는 단순 UI 결함이 아니라 의사결정 왜곡으로 이어질 수 있음
- 이후 UI 개선, 장중 품질 표시, artifact 안정화 작업도 이 기준 위에서만 일관되게 진행 가능
- 현재 코드베이스는 이미 장중 컨텍스트와 장마감 레짐이 분리돼 있으므로, 이제 제품 레이어가 그 분리를 제대로 드러내야 함

즉 이 티켓은 “새 기능 추가”보다 `제품 의미 정합성 복구`에 가깝다.

## 3. 목표

목표:

- 모든 핵심 화면에서 `이 값이 언제 기준인지`를 사용자가 해석 없이 바로 알 수 있게 한다.
- 장중 분위기, 장마감 시장국면, 다음 거래일 추천 계획을 절대 같은 의미로 보여주지 않는다.
- 장중 데이터가 약할 때는 “판단 보류”를 먼저 보여주고, fallback은 fallback이라고 명시한다.
- 이후 모든 화면/리포트/Discord/UI helper가 동일한 시간축 규칙을 따른다.

## 4. 비목표

이 티켓의 비목표:

- 실시간 시세 시스템 자체를 새로 구축하는 것
- 장중 tick/websocket 수집 추가
- 장중 정책/메타모델 로직 자체의 성능 개선
- 추천 알고리즘 변경
- 자동매매 연결

이 티켓은 `의미 분리와 UI/데이터 표시 리팩토링`이 핵심이다.

## 5. 핵심 용어 정의

이 티켓에서는 다음 용어를 강제로 구분한다.

### 5.1 장중 분위기

- 오늘 장중 체크포인트 기준
- 입력은 `fact_intraday_market_context_snapshot`
- 예: `장중 강세`, `장중 약세`, `장 시작 전`, `장중 데이터 보강 중`

### 5.2 일간 시장국면

- 특정 거래일 종가 기준 일간 레짐
- 입력은 `fact_market_regime_snapshot`
- 예: `3월 11일 종가 기준 상승 우위 장`

### 5.3 장마감 추천

- 특정 `as_of_date` 장마감 후 생성된 추천/선정 결과
- 입력은 `fact_ranking`, `fact_prediction`, `fact_selection_outcome`

### 5.4 다음 거래일 계획

- 다음 진입일용 목표북/리밸런스/추천 구성안
- 입력은 `fact_portfolio_target_book`, `fact_portfolio_rebalance_plan`

### 5.5 fallback

- 원래 보여줘야 하는 축의 데이터가 없거나 약해서, 다른 축의 마지막 유효 값을 대신 보여주는 상태
- fallback은 정상처럼 보이면 안 되고, 항상 fallback임을 노출해야 한다

## 6. 현재 문제 사례

### 6.1 홈

- `시장 국면` 또는 `장 분위기`가 실제로는 전일 종가 레짐인데 현재 장 상태처럼 보일 수 있음
- `최신 장중 세션`과 `현재 분위기`가 서로 다른 기준인데 연결 설명이 약함

### 6.2 시장현황

- 장중 사용자가 보는 첫 화면인데, 장마감 레짐과 장중 컨텍스트가 혼합되어 읽힘
- 기준일/기준시각 표시가 부족함

### 6.3 종목분석

- 즉석 추천 계산과 마지막 장마감 추천 결과가 함께 나오지만, 둘의 기준 축이 분명히 구분되지 않음

### 6.4 장중콘솔

- 장중 데이터 품질이 약할 때도 사용자는 “지금 판단 가능한 상태”로 오해할 수 있음

### 6.5 리포트/Discord

- 장마감 보고서, 장중 브리프, 사후평가가 모두 요약 메시지로 보일 수 있어 시간축 오해 가능

## 7. 리팩토링 원칙

### 7.1 모든 핵심 값에는 기준 축이 있어야 한다

핵심 값:

- 시장 분위기
- 시장국면
- 추천 점수
- 추천 여부
- 목표가/손절선
- 추천 구성안
- 포트폴리오 진입 상태
- 장중 액션

이 값들에는 최소한 다음 중 하나가 반드시 붙어야 한다.

- `장중 10:00 기준`
- `3월 11일 종가 기준`
- `3월 11일 장마감 배치 기준`
- `3월 12일 진입 예정안`

### 7.2 fallback은 숨기지 않는다

허용되는 표현:

- `오늘 장중 데이터가 약해 전일 종가 기준 흐름을 함께 봅니다.`
- `장중 데이터 보강 중`
- `직전 장 국면은 상승 우위 장입니다.`

허용되지 않는 표현:

- 장중 데이터가 없는데 그냥 `상승 우위 장`
- 전일 기준인데도 `현재 장 분위기`

### 7.3 장중 축과 장마감 축은 카드 단위로도 분리한다

같은 카드 안에 섞지 않는다.

예:

- 카드 A: `현재 장 분위기`
- 카드 B: `직전 장 종가 기준 시장국면`
- 카드 C: `다음 거래일 추천안`

### 7.4 실시간 축은 “판단 가능 여부”를 먼저 보여준다

장중에는 상태가 우선이다.

예:

- `장중 강세`
- `장중 혼조`
- `장중 약세`
- `장 시작 전`
- `장중 데이터 보강 중`

그 다음에 근거를 붙인다.

## 9. 구현 요구사항

### 9.1 공통 helper 계층

새로운 공통 helper 계층을 둔다.

필수 역할:

- 시간축 판별
- 기준 라벨 생성
- fallback 여부 표시
- 장중 데이터 품질/커버리지 해석

권장 반환 구조:

```python
{
    "mode": "intraday|intraday_stale|preopen|daily|missing",
    "headline": "...",
    "label": "...",
    "detail": "...",
    "reference_date": ...,
    "reference_time": ...,
    "fallback_used": bool,
}
```

현재 일부 반영된 `latest_market_mood_summary()`는 이 티켓의 시작점으로 간주한다.
최종적으로는 이 helper가 홈/시장현황/기타 화면의 공통 기준이 되어야 한다.

### 9.2 홈 화면 요구사항

- 상단 핵심 metric 중 하나는 `현재 장 분위기`
- 이 값은 장중 컨텍스트 우선
- 장중 불가 시 `전일 종가 기준`을 명시한 fallback
- `현재 기준 요약` 카드에도 장 분위기 기준 라벨 포함
- `최신 장중 세션` 같은 간접 값보다 `지금 판단 가능한지`가 먼저 보이게 조정

### 9.3 시장현황 화면 요구사항

- 장중 분위기 카드와 일간 시장국면 카드를 분리
- `시장 요약` 문장은 장중 기준으로 우선 작성
- 장중 fallback이면 문장 안에 이유를 포함
- `시장 국면 / 수급` 표는 일간 레짐과 장중 컨텍스트를 섞지 않음

### 9.4 종목분석 화면 요구사항

- `즉석 추천 계산`
  - 실시간/현재 기준
- `종목 핵심 요약`
  - 마지막 장마감 기준
- `최신 포트폴리오 진입 상태`
  - 다음 거래일 계획 기준

이 세 블록의 기준 축이 서로 다르면, 카드 제목 또는 보조문구에 반드시 표시한다.

### 9.5 장중콘솔 요구사항

- 현재 체크포인트
- 커버리지 상태
- 장중 분위기 판정 가능 여부

위 세 가지를 화면 가장 위에서 먼저 보여준다.

### 9.6 문구 규칙

허용 예:

- `3월 12일 10:00 기준 장중 강세`
- `3월 11일 종가 기준 상승 우위 장`
- `3월 11일 장마감 배치 기준 추천`
- `3월 12일 진입 예정안`

금지 예:

- `현재 시장국면: 상승 우위 장`  
  이유: 현재가 장중인지, 전일 종가 기준인지 알 수 없음

## 10. 데이터 판정 규칙

### 10.1 장중 우선 규칙

오늘 날짜의 `fact_intraday_market_context_snapshot`가 있고:

- `context_scope == market`
- 충분한 coverage
- `market_session_state != planned`
- `data_quality_flag != weak`

이면 장중 기준을 우선 사용한다.

### 10.2 장전 규칙

오늘 장중 session은 있으나 아직 장 시작 전이면:

- `장 시작 전`
- `전일 종가 기준 흐름을 함께 봐야 한다`

로 보여준다.

### 10.3 장중 데이터 약함 규칙

오늘 장중 session은 있으나 coverage가 낮으면:

- `장중 데이터 보강 중`
- fallback을 사용하더라도 `직전 장 국면은 ...` 식으로 보조만 한다

### 10.4 fallback 규칙

오늘 장중 데이터를 확정 상태로 쓸 수 없을 때만 일간 레짐 fallback 허용.

fallback 표현은 반드시:

- `3월 11일 종가 기준`
- 또는 `직전 장 국면`

중 하나를 포함해야 한다.

## 13. 완료 기준

이 티켓은 아래를 모두 만족해야 완료다.

- 홈에서 현재 장 분위기와 전일 종가 레짐이 더 이상 혼동되지 않는다.
- 시장현황 첫 카드가 장중 기준인지 종가 기준인지 명확하다.
- 종목분석에서 즉석 추천/장마감 추천/다음 거래일 계획이 구분된다.
- 장중 데이터가 약할 때 화면이 확정적 톤으로 말하지 않는다.
- fallback은 사용자에게 숨기지 않는다.
- 관련 테스트가 추가된다.
- 서버 반영 후 실제 화면 문구가 기준 라벨과 함께 읽힌다.

## 14. 현재 상태

- 일부 선행 작업 반영됨
  - 장중 컨텍스트 우선 helper 초안 존재
  - 홈/시장현황에 일부 반영
- 그러나 제품 전체의 시간축 분리는 아직 미완료

즉 이 티켓은 “새로 시작”이 아니라, 이미 일부 들어간 작업을 기준 문서 아래에서 끝까지 정리하는 리팩토링 티켓이다.

## 15. 구현 메모

- 장중 분위기와 일간 시장국면은 절대 동일 레이블로 합치지 않는다.
- `현재`라는 단어는 장중 데이터가 실제 현재 축일 때만 사용한다.
- `추천`, `시장국면`, `분위기`, `진입 예정안`은 서로 다른 축의 명사이므로 제목 단계부터 분리한다.
- 이 티켓 이후의 UI 개편은 모두 이 시간축 규칙 위에서만 진행한다.
